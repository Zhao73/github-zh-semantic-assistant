# Chrome Web Store Listing Draft

## Name

GitHub 中文语义助手

## Short Description

Translate GitHub UI, README, issues, and pull requests into precise Chinese with protected names and AI terminology.

## Detailed Description

GitHub 中文语义助手 is a Chrome extension for Chinese-speaking developers who read GitHub every day.

It does not depend on browser machine translation. GitHub UI labels are translated locally with a curated dictionary, while long-form repository content is translated through your OpenAI-compatible model with a strict technical glossary.

What it handles:

- GitHub UI labels such as Pull requests, Issues, Actions, Releases, Fork, Star, Watch, Code scanning, and Dependabot alerts
- Repository descriptions, README sections, issue titles, issue comments, and pull request discussions
- AI-era terminology such as agent, agentic workflow, RAG, embedding, tool calling, MCP, context window, fine-tuning, and hallucination
- Proper names such as usernames, repository names, package names, model names, product names, and organizations

Why developers use it:

- Read GitHub faster without losing the original technical meaning
- Keep names like `openai/openai-cookbook`, `LangChain`, `Qwen2.5`, and `React` untouched
- Customize the glossary for your team, framework, paper, or internal project
- Use local OpenAI-compatible endpoints when you do not want to send content to hosted APIs
- Toggle the extension with `Alt+Shift+G` and translate the current page with `Alt+Shift+T`

Privacy:

- API keys stay in Chrome local storage
- Code blocks, commands, package names, paths, and protected names are preserved
- Long text is sent only to the model endpoint you configure
